# アプリ要件定義書：ペア向け夜間ルーティン管理アプリ「Sleep Sync」

## 1. アプリの目的・コンセプト

* **目的**: パートナーと夜のルーティンを共有し、お互いの生活リズムを尊重しながら、質の高い睡眠とデジタルデトックスを促す。
* **ターゲットデバイス**: iPhone（Safariブラウザでの利用、またはホーム画面に追加するPWA形式を想定したモバイル最適化UI）。
* **コンセプト**: 「起きる時間」からの逆算で夜のスケジュールを生成。目に優しいダークトーンと絵本のような温かみのある世界観で、入眠前のリラックスタイムを演出する。

## 2. システム要件（前提条件）

* **フロントエンド完結**: データベース（バックエンド）は使用せず、フロントエンド（HTML/CSS/JavaScript）のみで動作する。
* **データ保存**: ユーザー設定やタスク状態は、ブラウザのローカルストレージを使用してiPhone端末内に保持する。
* **パートナー連携**: URLパラメータの共有やQRコード読み取りなど、フロントエンド技術を用いて擬似的にスケジュールを共有する。

## 3. コア機能要件

### 3-1. 初期設定（オンボーディング）

* **ユーザー情報登録**: 初回起動時に「ユーザー名」「アイコン」と「目標睡眠時間」を設定する。
* **設定の永続化**: ここで設定した情報はローカルストレージに保存され、翌日以降のスケジュール生成時にも引き継がれる。

### 3-2. スケジュール逆算＆タスク生成

* **毎日の時間設定**: 「翌朝起きる時間」と「寝る時間」を設定すると、夜のタスクが逆算して自動スケジュールされる。
* **デフォルトタスク**:
* お風呂に入る（寝る90分前）
* お風呂をいれる（お風呂に入る15分前）


* **タスクの追加・変更**:
* 設定画面から任意のタスクを追加、および時間の変更が可能。
* 変更内容は保存され、翌日以降のスケジュールにも反映される。



### 3-3. デジタルデトックス設定

* **表示タイミング**: 起床・就寝時間を設定後、最優先で設定画面を表示する。
* **時間設定**: デフォルトは「寝る2時間前」。ユーザーの都合に合わせて時刻の調整が可能。お風呂に入る時間と合わせてタイムライン上に表示される。
* **ランダムメッセージ機能**: デジタルデトックス設定画面には、detox-messages.md で指定しておいた複数のメッセージの中から、ランダムで1つを選んで表示する。

### 3-4. タイムラインUIとパートナー共有

* **デュアルタイムライン表示**: 自分とパートナーのスケジュールが、同じタイムライン上で左右に分かれて表示される（例：左が自分、右がパートナー）。
* **スケジュールの連動**: デジタルデトックス開始時間や、お風呂に入る時間、就寝時間などが同じ時系列上で視覚的に比較・把握できる。

### 3-5. タスク進行と催促（カウントダウン）

* **カウントダウン表示**: 未完了タスクのうち、一番予定時刻が早いものに対して「あと1時間」「あと◯分」といったカウントダウンをリアルタイムで表示する。
* **超過時のアラート**: 予定時間を過ぎたタスクは、カウントダウンやタスク名の文字色を「赤文字」に変更し、視覚的に実行を促す（催促）。

* **通知方式**: タスクの催促・アラートや重要な状態変化は、ユーザーの注意を引くためにポップアップで表示する（ブラウザのNotification APIによるネイティブ通知と、環境に応じたアプリ内モーダル／トーストの両対応を想定）。

### 3-6. 完了アクションとねぎらいポップアップ

* **タスク完了時の反応**: タスクを完了（チェック）するごとに、ポップアップでねぎらいの言葉を表示する。
* **条件分岐メッセージ**:
* 予定時刻通り/早かった場合：「おつかれさま！」「いいペースだね！」など。
* 予定時刻を過ぎていた場合：「遅かったけどおつかれ！」「よくがんばったね！」など、状況に合わせたメッセージに変化する。



### 3-7. 就寝モードと日次リセット

* **「おやすみ」モード**: 最後の「就寝タスク」を完了すると、画面全体に大きめに「おやすみ」というメッセージが表示される。
* **操作ロック**: 就寝モード中は他のタスクの操作・閲覧ができなくなる（デジタルデトックスの強制）。誤操作や修正のための「戻る」ボタンのみ画面端に配置しておく。
* **起床後の自動リセット**: 設定した「起きる時間」を過ぎると、自動的にその日の「起きる時間・寝る時間の設定画面」に戻り、新しい1日のサイクルが始まる。

## 4. UI/UX・デザイン要件

* **トーン＆マナー**: 目に優しいダークな色合い（ミッドナイトブルーなど）。AIっぽさ（無機質・サイバー感）を排除した、絵本やアナログ感のある落ち着いた温かいトーン。
* **iPhone最適化**: 画面下部のセーフエリアへの配慮や、スワイプ操作に馴染むUI設計。
* **時刻設定UI**: 朝起きる時間や寝る時間の設定は、アナログ時計の文字盤をなぞるような直感的でアナログ感のあるインターフェースを採用。

## 実装メモ（通知／ポップアップ）

* **ネイティブ通知（推奨）**: ブラウザ環境では `Notification API` を利用し、許可が得られた場合はネイティブのプッシュ風ポップアップで催促を表示する。iOSのPWA制約を考慮し、Safariでの動作はアプリ内モーダルにフォールバックする。
* **フォールバック（必須）**: Notification API が利用不可または権限がない場合は、画面内のモーダル／トーストで同等の通知を表示する（視認性の確保と操作ブロッキングのオプションを設ける）。
* **完了時のポップアップ**: タスク完了時のねぎらいメッセージは短いポップアップで表示し、自動で閉じるかユーザーが閉じるまで残す設定を提供する。

* **実装上の注意**: iOS Safari ではブラウザ通知が制限されるため、ホーム画面追加（PWA）含めた挙動確認を行う。ローカルストレージに通知既読やスヌーズ設定を保存して、同一端末内での一貫性を保つ。
